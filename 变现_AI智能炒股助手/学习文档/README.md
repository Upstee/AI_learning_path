## 📓 Jupyter Notebook使用规则

### 核心原则

**所有包含代码和Markdown混合内容的知识点，必须使用Jupyter Notebook（.ipynb）格式**

### 适用场景

#### 必须使用.ipynb的情况：
1. **技术分析知识点**（K线图、技术指标、图表形态等）
   - 需要绘制图表
   - 需要运行代码示例
   - 可以边学边运行

2. **量化交易知识点**（策略开发、回测、因子分析等）
   - 包含代码示例
   - 需要运行验证
   - 理论与实践结合

3. **AI应用知识点**（预测模型、情感分析、图像识别等）
   - 包含模型代码
   - 需要运行示例
   - 可以交互式学习

4. **基本面分析中的图表**（财务报表可视化、估值模型等）
   - 需要绘制图表
   - 需要计算示例
   - 可以交互式学习

#### 使用.md的情况：
1. **纯理论知识**（如股票市场基本概念、交易规则等）
   - 不需要代码
   - 不需要图表
   - 纯文字说明

### Jupyter Notebook结构要求

每个.ipynb文件应包含：
1. **Markdown单元格**：理论知识讲解
2. **代码单元格**：可运行的代码示例
3. **输出单元格**：代码执行结果（图表、数据等）
4. **注释说明**：代码的详细注释

### 使用优势

1. **交互式学习**：可以边看边运行，加深理解
2. **即时反馈**：运行代码立即看到结果
3. **易于修改**：可以修改参数，观察不同结果
4. **便于分享**：可以导出为HTML、PDF等格式

### 工具准备

**安装Jupyter Notebook**：
```bash
pip install jupyter notebook
```

**启动Jupyter Notebook**：
```bash
jupyter notebook
```

**或使用JupyterLab**（推荐）：
```bash
pip install jupyterlab
jupyter lab
```

### 注意事项

1. ✅ **代码要可运行**：确保代码可以直接运行，不依赖外部复杂环境
2. ✅ **数据要可获取**：使用公开数据或提供数据获取方法
3. ✅ **注释要详细**：代码要有详细的中文注释
4. ✅ **结果要展示**：重要结果要展示在输出单元格中
5. ⚠️ **依赖要说明**：在文件开头说明需要的Python包
